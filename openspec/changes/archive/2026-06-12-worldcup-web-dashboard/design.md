## Context

当前世界杯预测数据存储在两个Markdown文件中：
- 小组赛【第一轮】AI 预测跑分看板.md - 包含各模型的预测数据
- 小组赛【第一轮】AI预测结果汇总.md - 包含汇总表格和比赛结果

用户选择了赛事主题风格（深色背景 + 荧光绿/金色点缀），需要将这些数据转化为两个专业的Web展示页面。

## Goals / Non-Goals

**Goals:**
- 创建两个独立的HTML页面，分别展示预测看板和结果汇总
- 采用深色主题设计，荧光绿(#00ff88)和金色(#ffd700)作为强调色
- 实现响应式布局，支持桌面和移动端
- 数据以JSON格式内嵌，便于维护和更新
- 添加平滑动画和交互效果

**Non-Goals:**
- 不需要后端服务器或数据库
- 不需要实时数据更新功能
- 不需要用户登录或权限管理
- 不需要复杂的图表库（使用原生CSS实现可视化）

## Decisions

### 1. 技术栈选择
**决策**: 纯HTML + CSS + JavaScript，无框架依赖

**理由**:
- 项目规模小，不需要React/Vue等框架
- 便于部署为静态页面（GitHub Pages/Gitee Pages）
- 加载速度快，无构建步骤
- 用户可直接打开HTML文件查看

**替代方案**:
- React + Vite: 过于复杂，增加构建步骤
- Vue: 同样需要构建工具

### 2. 数据存储方式
**决策**: 将Markdown数据转换为JSON，内嵌在HTML文件中

**理由**:
- 避免跨域请求问题
- 简化部署，单文件即可运行
- 数据量小（约4场比赛，4个模型），JSON体积可控

**替代方案**:
- 外部JSON文件: 需要HTTP服务器
- LocalStorage: 增加复杂度，不适合静态展示

### 3. 样式实现方案
**决策**: 使用CSS变量 + 媒体查询实现响应式

**理由**:
- CSS变量便于主题色管理
- 媒体查询实现移动端适配
- 使用backdrop-filter实现毛玻璃效果
- CSS Grid/Flexbox实现灵活布局

### 4. 页面结构设计
**决策**: 两个独立页面，共享导航栏

**理由**:
- 预测看板和结果汇总功能独立
- 共享导航栏提供一致的用户体验
- 便于后续扩展更多页面

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 数据更新需要手动修改JSON | 中 | 提供清晰的数据格式说明，便于维护 |
| 移动端表格展示可能拥挤 | 低 | 使用卡片式布局替代表格，或横向滚动 |
| 浏览器兼容性（backdrop-filter） | 低 | 提供降级方案，确保基础功能可用 |
| 页面加载大量数据时卡顿 | 低 | 当前数据量小，无需优化；未来可考虑分页 |

## Migration Plan

1. 解析现有Markdown文件，提取预测数据
2. 转换为JSON格式，验证数据完整性
3. 创建HTML页面结构和样式
4. 实现数据渲染逻辑
5. 测试响应式布局和交互效果
6. 部署为静态页面
